iOS中音频后台播放的实现与锁屏界面的控制

How to handle background audio playing while iOS device is locked or on another application?

iOS中实现音频的后台播放

最近因为一个项目优化的事情需要完成听书音频后台的持续播放，和锁屏界面的控制，网上查了一些资料，都是属于比较老的，也是遇到了不少坑，自己记录一下遇到的一些问题和解决方法。

首先添加支持库和系统配置

我查到的首先是需要在Info.plist里加上UIBackgroundModes再到里面加上audio这个属性，我发现这个在现在iOS9并不需要，真正需要做的是到Capabilities下的BackgroundModes中打开Audio，AirPlay and Picture in Picture

如下图
;

支持库呢就没太大出入，一共三个支持库:

  • MediaPlayer.framework
  • CoreMedia.framework
  • AVFoundation.framework

到这里准备工作就做完了，下面上具体代码

本就音乐文件后台播放的实现

首先当然是在AppDelegate里加入#import 框架然后：

- (void)applicationWillResignActive:(UIApplication *)application {

//将要进入后台
//开启后台处理多媒体事件
[[UIApplication sharedApplication] beginReceivingRemoteControlEvents];
AVAudioSession *session=[AVAudioSession sharedInstance];
[session setActive:YES error:nil];
//后台播放
[session setCategory:AVAudioSessionCategoryPlayback error:nil];
}

这样就能实现最简单的本地音频后台播放，但是不能持续播放网络音频和锁屏控制（即远程控制），我们一个一个来实现

持续播放网络音频

若需要持续播放网络歌曲，还需要申请后台任务id，具体做法是:

    //首先给AppDelegate添加一条TaskId属性
    @property (nonatomic,assign) UIBackgroundTaskIdentifier FGTaskId;


    //自定义一个backgroundPlayerID方法如下：

    + (UIBackgroundTaskIdentifier)backgroundPlayerID:(UIBackgroundTaskIdentifier)backTaskId {

//设置并激活音频会话类别
AVAudioSession *session=[AVAudioSession sharedInstance];

[session setCategory:AVAudioSessionCategoryPlayback error:nil];
[session setActive:YES error:nil];

//设置后台任务ID
UIBackgroundTaskIdentifier newTaskId=UIBackgroundTaskInvalid;
newTaskId=[[UIApplication sharedApplication] beginBackgroundTaskWithExpirationHandler:nil];
if(newTaskId!=UIBackgroundTaskInvalid&&backTaskId!=UIBackgroundTaskInvalid)
{
    [[UIApplication sharedApplication] endBackgroundTask:backTaskId];
}
return newTaskId;
}

然后在applicationWillResignActive中调用我们的方法

- (void)applicationWillResignActive:(UIApplication *)application {

//将要进入后台
//开启后台处理多媒体事件
[[UIApplication sharedApplication] beginReceivingRemoteControlEvents];
AVAudioSession *session=[AVAudioSession sharedInstance];
[session setActive:YES error:nil];
//后台播放
[session setCategory:AVAudioSessionCategoryPlayback error:nil];

//需要持续播放网络歌曲，还需要申请后台任务id，具体做法如下:

_bgTaskId = [AppDelegate backgroundPlayerID:_bgTaskId];

//其中的_bgTaskId是后台任务UIBackgroundTaskIdentifier _bgTaskId;

}

这样我们就实现网络音频的后台播放啦~

锁屏控制

需要实现远程控制则要在你的播放控制器实现remoteControlReceivedWithEvent这个方法，具体如下：

    #pragma mark 后台的一些处理操作
- (void)remoteControlReceivedWithEvent:(UIEvent *)event
{
if (event.type == UIEventTypeRemoteControl) {

    if (event.subtype == UIEventSubtypeRemoteControlPlay || event.subtype ==UIEventSubtypeRemoteControlPause) {

        //暂停开始按钮调用方法
        [[FGPlayNews sharedFGPlayNews] startToPlay];

    } else if (event.subtype == UIEventSubtypeRemoteControlNextTrack){
        //下一首按钮调用方法
        [[FGPlayNews sharedFGPlayNews] nextMusic];


    }else if(event.subtype == UIEventSubtypeRemoteControlPreviousTrack)
    {
        //上一首按钮调用方法
        [[FGPlayNews sharedFGPlayNews] preMusic];
    }

}
}

然后还要在AppDelegate的applicationDidEnterBackground实现允许远程控制的代码如下：

    - (void)applicationDidEnterBackground:(UIApplication *)application {
//进入后台
[application beginReceivingRemoteControlEvents];

}

现在就能实现网络歌曲后台播放和锁屏的时候远程控制啦~当然还没完，我们在锁屏下并不知当前播放的音频名字和图片，如果需要加入名字和图片则需要实现configNowPlayingInfoCenter方法代码如下，当然首先要添加头文件 #import ：

    MPMediaItemArtwork * mArt = [[MPMediaItemArtwork alloc] initWithImage:self.image];

    [MPNowPlayingInfoCenter defaultCenter].nowPlayingInfo = nil;

    [[MPNowPlayingInfoCenter defaultCenter] setNowPlayingInfo:dict];

    }

    */

在configNowPlayingInfoCenter的方法实现了之后，在每次换音频的时候都需要重新调用一次这个方法，刷新信息，差不多到这里基本功能都实现了，具体更深入的可以找博主探讨~

runTime之关联

runTime初尝试—-关联

关联

顾名思义就是将一个对象作为另一个对象的一部分，将他两动态的关联起来。好处就是不需要修改类的定义，就能为其增加储存空间。关联是基于key来绑定对象的，理论上是可以绑定无限个对象的。

创建关联（objc_setAssociatedObject）

创建关联需要使用object-c运行时的函数，所以首先需要添加#import 头文件，然后使用objc_setAssociatedObject函数，其中有4个参数，下面上代码，将一个字符串关联到数组上：

//array为关联的源对象
NSArray *array =@[@"One", @"Two", @"Three"];
//overview为关联的对象
NSString * overview = [[NSString alloc] initWithFormat:@"%@",@"First three numbers"];
//@"wfg"为关联的key，即关键字。
//OBJC_ASSOCIATION_COPY_NONATOMIC为关联策略，关联策略表明了相关的对象是通过赋值，保留引用还是复制的方式进行关联的；还有这种关联是原子的还是非原子的，这个关联策略和声明属性时很类似。因为这个我关联的对象是nsstring所以用的copy
objc_setAssociatedObject(array, @"wfg", overview, OBJC_ASSOCIATION_COPY_NONATOMIC);

获取关联对象（objc_getAssociatedObject）

获取源对象上key对应的关联对象接上面的代码：

//objc_getAssociatedObject返回的对象为id型，所以需要强转为自己需要的类型
//两个参数，分别为源对象和key
NSString *associatedObject = (NSString *)objc_getAssociatedObject(array, @"wfg");

断开关联

关联的断开，分成两种：

  • 部分关联的断开，用objc_setAssociatedObject方法将你想断开的key对应的关联对象设置为nil
  • 所有关联的断开，即对象关联状态的重置，使用objc_removeAssociatedObjects就可使对象初始到最开始的未关联状态。

wfgmarkDown

MarkDown的基本用法

标题的设置

在markDown中有两种方法设置标题

  • 第一种：在文字前面加上“#”，通过“#”的数量级来代表标题的关系。

  • 第二种：在文字下面加上“+”和“-”，分别表示一级和二级(最好三个以上)。

块注释

通过在文字开头添加“>”表示块注释。（当>和文字之间添加五个blank时，块注释的文字会有变化。）

斜体

将需要设置为斜体的文字两端使用1个“*”或者“_”夹起来

粗体

将需要设置为斜体的文字两端使用2个“*”或者“_”夹起来

无序列表

在文字开头添加(*, +, and -)实现无序列表。但是要注意在(*, +, and -)和文字之间需要添加空格。（建议：一个文档中只是用一种无序列表的表示方式）

有序列表

使用数字后面跟上点号.（还要有空格）

超链接

Markdown中有两种方式，实现链接，分别为内联方式和引用方式。

  • 内联方式：This is an [example link](http://example.com/).
  • 引用方式：
    I get 10 times more traffic from [Google][1] than from [Yahoo][2] or [MSN][3].

[1]: http://google.com/ “Google”

[2]: http://search.yahoo.com/ “Yahoo Search”

[3]: http://search.msn.com/ “MSN Search”

图片

图片的处理方式和链接的处理方式，非常的类似。

  • 内联方式：![alt text](/path/to/img.jpg “Title”)
  • 引用方式：![alt text][id]

[id]: /path/to/img.jpg “Title”

代码

实现方式有两种：

  • 第一种：简单文字出现一个代码框。使用`\`。（`不是单引号而是左上角的ESC下面~中的`）
  • 第二种：大片文字需要实现代码框。使用Tab和四个空格。
脚注

实现方式如下：
hello[^hello]

[^hello]: hi

下划线

在空白行下方添加三条“-”横线。（前面讲过在文字下方添加“-”，实现的2级标题）
